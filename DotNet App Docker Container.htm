Step #2 - Create Docker Image

 

Create the docker file

example 1:

 

1) Get the base image

 

FROM  mcr.microsoft.com/dotnet/runtime:6.0

 

2) Copy all published files to image working directory

 

COPY  bin/Release/net6.0/publish .

 

3) Set the entry point of Docker image

 

ENTRYPOINT [ "dotnet" , "DockerConsoleAppDemo.dll" ]

 

 

Example 2:

 

 

 

1) Get the base image

2) Set Working directory in the base image

3) Copy *.* to working directory /app

4) Run the build command inside the docker image with -c (solution configuration)  of “Release”

5) Set the entry point to specify the command when the container is started.  In this case, that is dotnet “run” with the no launch profile flag

 

Note: In this case, because we haven't specified very explicitly what files we wanted to be into this docker image so all of the files under our project directory will have got copied and that will include things like the launch Settings JSON file. by default, asp.net core 2-0 will now look at that launch settings file and use the port and environment from that file. If it finds it in this case, we don't want to do that. Once, this image is running. I want it to be running on the default asp.net core port or 80. So by adding the no launch profile flag, I can avoid that launch settings file and its values being used.

 

 

 

Build the image

 

Example 1

 

> Docker build -t  dockerdotnet_image .

 

Note: (.) is the path of the build context. Build context specifies all of the files that we want to potentially make up our build.

 

(--tag) = -t

 

Note: docker image must be lower case.

 

 

Step #3 - Run Docker Image

 

 

 

> docker run --rm -p 6061:80 dockerdotnet_image

> docker run --rm -p 6061:80 dockerdotnet_image

> docker run --rm -p 6061:80 -e ASPNETCORE_URLS=http://+:80 dockerdotnet_image

 

 

Note: Publish the port 80 of the container to 6001 on my host.  That's needed so that we can actually communicate with the application that's running inside that container which is listening on port 80. If we didn't do that, we wouldn't have a way to pass the address from our localhost through into the container.

     image: ${DOCKER_REGISTRY-}backend

     build:

       context: .

       dockerfile: backend/Dockerfile

     environment:

      - ASPNETCORE_ENVIRONMENT=Development

      - ASPNETCORE_URLS=https://+;http://+

      - ASPNETCORE_HTTPS_PORT=5803

       #ASPNETCORE_URLS: "https://+;http://+"

       #ASPNETCORE_HTTPS_PORT: "5803"

       #ASPNETCORE_ENVIRONMENT: "Development"

      - "5800:80"

      - "5803:443"

     volumes:

      - ${APPDATA}\Microsoft\UserSecrets\:/root/.microsoft/usersecrets

      - ${USERPROFILE}\.aspnet\https\:/root/.aspnet/https/
