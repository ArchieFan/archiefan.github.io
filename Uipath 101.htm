Read data from Config File

 

 

 

 

Prefixing the string with an @ indicates that it should be treated as a literal, i.e. no escaping.

For example if your string contains a path you would typically do this:

 

string path = "c:\\mypath\\to\\myfile.txt";

 

The @ allows you to do this:

 

string path = @"c:\mypath\to\myfile.txt";

 

Notice the lack of double slashes (escaping)

How to Connect Orchestrator to UiPath assistant

 

1.  You can use either client ID and Machine Key to connect orchestrator.  First, you need to create Tenants.

 

 

 

2. Then go to orchestrator -> Tenants -> Machines tab.  Add the new machines.

 

 

 

3. Using the Machine Name in UiPath assistant to create the new machines.  Or you can get it from CMD by type in hostname

 

 

 

 

4. You will have the machine key and client ID/ secret.

 

 

5. Copy the machine Key and URL (just copy the URL) to UiPath assistant.  Then click connect.

6. After connected to Orchestrator, you will need to assist the license to it.

 

 

 

7. You can see that Machine is ready and version status is Compliant.

 

 

 

8. Go to Folder tab and create new folder.

 

 

9. In the new created folder, you assign the roles and Robots to this Users.

 

 

 

10. Assign all roles

 

 

 

11. In the Robot setup page, you enable both “Attended Robot” and “Unattended Robot”.

Add items to Queue

 

 

Unique Reference - Enables you to enforce unique references in all the transactions that are to be added to the queue, by selecting the Yes button. If the No button is selected, then the queue can contain transactions with duplicate references.

 

Add queue item - Adding the item into the queue

 

 

 

Get Queue Item from Orchestrator

 

Get transaction Item - Get an item form the queue, so that we can process it further.  It sets the status of the item to progress in Orchestrator.

 

 

 

Set Transaction Item - Used to set the status of transaction/queue item to failed or successful inside Orchestrator.

 

 

 

 

 

 

 

UiPath ReFramework (Robotic Enterprise Framework) Overview 

 

It is a template used to build, test, and run robots at scale. Advantages: logging, scalability, reusability, and error handling.

 

You can never build anything if you don’t understand what’s going on. That’s why process descriptions are very very important.

 

ReFranework is all about queues and transaction.  The best practice is to split it into dispatcher and performer.
