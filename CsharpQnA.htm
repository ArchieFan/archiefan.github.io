Q9.1. Mutable Vs immutable types?

C# is a statically typed programming language that supports both mutable and immutable types. Immutable types are those whose values cannot be changed once they are created, while mutable types are those whose values can be changed after they are created.

 

Immutable Types:

 

⚫ Once created, the values of immutable types cannot be changed. Any operation that appears to modify the value of an immutable object actually creates a new object with the updated value.

⚫ Immutable types are typically used for objects whose state should not change after they are created, such as values that represent constants, settings, or configurations.

⚫ Immutable types are inherently thread-safe, as they can be shared among multiple threads without the need for locks or synchronization.

⚫ Examples of immutable types in C# include built-in value types such as int, float, and char, as well as user-defined types created with the readonly modifier or with properties that have only getter methods.

 

Mutable Types:

 

⚫ Mutable types, on the other hand, allow their values to be changed after they are created. This means that the state of an object can be modified at any time, leading to potential changes in behavior or unexpected consequences.

⚫ Mutable types are typically used for objects whose state needs to be updated frequently, or whose behavior depends on the changing state of the object, such as objects representing real-time data, user input, or database entities.

⚫ Mutable types require careful management of state changes to avoid issues such as race conditions and unintended side effects, especially in multi-threaded environments.

⚫ Examples of mutable types in C# include built-in reference types such as arrays, lists, and dictionaries, as well as user-defined types created with fields, properties with setter methods, or with methods that modify the state of the object.

 

In general, using immutable types can help to reduce complexity and improve maintainability of code, as they provide clear separation between data and behavior, and eliminate the need for explicit synchronization. However, mutable types can also be useful in certain scenarios where dynamic state changes are required. It's important to choose the appropriate type depending on the specific requirements of your code and ensure proper handling of state changes to avoid unintended consequences.

Generic Collections

Description

List<T>

Generic List<T> contains elements of specified type. It grows automatically as you add elements in it.

Dictionary<TKey,TValue>

Dictionary<TKey,TValue> contains key-value pairs.

SortedList<TKey,TValue>

SortedList stores key and value pairs. It automatically adds the elements in ascending order of key by default.

Queue<T>

Queue<T> stores the values in FIFO style (First In First Out). It keeps the order in which the values were added. It provides an Enqueue() method to add values and a Dequeue() method to retrieve values from the collection.

Stack<T>

Stack<T> stores the values as LIFO (Last In First Out). It provides a Push() method to add a value and Pop() & Peek() methods to retrieve values.

Hashset<T>

Hashset<T> contains non-duplicate elements. It eliminates duplicate elements.

 

Non-generic Collections

Usage

ArrayList

ArrayList stores objects of any type like an array. However, there is no need to specify the size of the ArrayList like with an array as it grows automatically.

SortedList

SortedList stores key and value pairs. It automatically arranges elements in ascending order of key by default. C# includes both, generic and non-generic SortedList collection.

Stack

Stack stores the values in LIFO style (Last In First Out). It provides a Push() method to add a value and Pop() & Peek() methods to retrieve values. C# includes both, generic and non-generic Stack.

Queue

Queue stores the values in FIFO style (First In First Out). It keeps the order in which the values were added. It provides an Enqueue() method to add values and a Dequeue() method to retrieve values from the collection. C# includes generic and non-generic Queue.

Hashtable

Hashtable stores key and value pairs. It retrieves the values by comparing the hash value of the keys.

BitArray

BitArray manages a compact array of bit values, which are represented as Booleans, where true indicates that the bit is on (1) and false indicates the bit is off (0).

Q34 - What is an Event? 

 

An event is a notification sent by an object to signal the occurrence of an action.The class who raises events is called Publisher, and the class who receives the notification is called Subscriber. There can be multiple subscribers of a single event. Typically, a publisher raises an event when some action occurred. The subscribers, who are interested in getting a notification when an action occurred, should register with an event and handle it.

 

Events are wrapper over delegates, they are encapsulation over delegates. they ensure delegates cannot be tempered by anyone.  Event change the delegate to a publisher subscriber model.

 

Events are encapsulation over delegate.

Q85. What is the use of Yield keyword in c# ?

 

 

 

 

 

 

 

 

 

Using yield makes the collection lazy.

Let's say you just need the first five items. Your way, I have to loop through the entire list to get the first five items. With yield, I only loop through the first five items.

 

IEnumerable<object> FilteredList()

{

    foreach( object item in FullList )

    {

        if( IsItemInPartialList( item ) )

            yield return item;

    }

}

 

IEnumerable<object> FilteredList()

{

    var list = new List<object>();

    foreach( object item in FullList )

    {

        if( IsItemInPartialList( item ) )

            list.Add(item);

    }

    return list;

}
