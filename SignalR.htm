SingalR

 

SignalR is a real-time communication library for building web applications. It enables server-side code to push content to connected clients in real-time. This is useful for scenarios where you want to update the user interface in real-time, for example, when you are building a chat application or a stock ticker.

To use SignalR, you need to create a SignalR server and a SignalR client. The server is a piece of code that runs on your server and manages the connections with the clients. The client is a piece of code that runs in the user's web browser and connects to the server.

Once the client and the server are connected, the server can send messages to the client, and the client can send messages to the server. The client and the server can also invoke methods on each other.

SignalR uses WebSockets to establish a persistent connection between the client and the server, but it also falls back to other technologies such as long polling if WebSockets are not available.

Overall, SignalR is a powerful tool for building real-time web applications, and it is easy to use with a simple and intuitive API.

Immediately Invoked Function Expression

 

An IIFE (Immediately Invoked Function Expression) is a JavaScript function that runs as soon as it is defined. It’s always the best to put everything inside of Immediately Invoked Function Expression. It can be used to avoid variable hoisting from within blocks, protect against polluting the global environment and simultaneously allow public access to methods while retaining privacy for variables defined within the function.
