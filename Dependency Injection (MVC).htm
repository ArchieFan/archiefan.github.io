1) What is need of Dependency Injection?

 

DI is a design methodology where in rather than caller creating the instance its injected by some framework or some other mechanism.

 

 

 

2) Explain IOC (Inversion of control)?

 

Inversion of control (IOC) is a principle where we say focus on your work and other tasks delegate to someone else. DI is implementing the IOC.

 

 

3) What is the benefit of DI?

 

DI helps to implement decoupled architecture where you change at one place and changes are reflected at many places.

 

4) Scoped vs Transient vs Singleton?

 

Scoped, Transient, and Singleton define the life time of the DI object.

 

Singleton :- only one instance of object is created for the whole web application.(like cache object)

 

the GUID is always the same in the whole web application.

 

 

Scoped

1. For every request, new instance will be injected.

2. In same request, if we DI multiple times same instance is injected.

 

For every request, new instance will be injected. but within a single request, no matter how many time you ask for object.  The same instance will be injected.

 

 

 

Transient

1. For every request, new instance will be injected.

2. In the same request if we DI multiple times different instance is injected.

 

For every request, new instance will be injected. but within a single request, if you ask for object many times.  The different instance will be injected.

 

 

 

5) When should we use what?

 

 

Singleton will be used for caching, Shared Service, Hit counter

 

Scoped :- Repository for same transaction, business object, used most of the time.  if change one instance of object, all instance will be changed.

 

 

 

Transient :- instance should not affect each other like different transaction, thread safety a object are different instances, same object asked multiple times in the constructor. (separate instance)
