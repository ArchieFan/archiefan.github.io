AWS S3 Static Hosting with React App and Cloudfront

 

 

I'm going to show you how to set up a react application that is hosted on s3 via CloudFront as a distribution it's going to involve creating some certificates and also some route 53 DNS modifications.

 

You're wondering why behind CloudFront. This is because you cannot install the free AWS certificate manager certificates directly onto your website or application.  Instead you must install your certificates by using one of the services integrated with ACM.  And these services are elastic load balancing, Amazon CloudFront, AWS elastic beanstalk, Amazon API gateway, and AWS CloudFormation.  So, in our case, I put one S3 instance with a basic web site behind a CloudFront and then applied the free ACM certificates on the CloudFront.

6.  Create Record

 

We are going to go to route 53 and this is basically where we set our domain registration or domain settings. So that they point when someone tries to go to my website. It's going to point to that actual index.html and load up our website.  What we need to do is create two different records. So that when someone tries to hit our website www.archiefan.co.uk or archiefan.co.uk.  In this case, they're going to get forwarded to the s3 buckets content.

 

 

 

 

 

 

 

 

l  CNAME (Canonical Name): A CNAME record creates an alias for a domain name. It allows you to point one domain name to another domain name. For example, if you have a CNAME record for "www.example.com" pointing to "example.com," when someone tries to access "www.example.com," they will be directed to "example.com."

 

l  A (Address): An A record maps a domain name to an IPv4 address. It is used to translate domain names into IP addresses. For instance, if you have an A record for "example.com" pointing to "192.0.2.1," when someone tries to access "example.com," their request will be directed to the server with the IP address "192.0.2.1."

 

l  SOA (Start of Authority): The SOA record indicates the authoritative DNS server for a domain. It contains information about the domain, such as the primary name server, the email address of the domain administrator, the domain serial number, and other timing parameters for zone refreshing and retrying.

 

l  NS (Name Server): NS records specify the authoritative name servers for a domain. They delegate the responsibility of DNS resolution for a subdomain to a specific name server. For example, if you have a subdomain "sub.example.com," you would have NS records that point to the name servers responsible for resolving queries for that subdomain.
