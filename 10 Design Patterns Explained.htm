Creational patterns

- how objects are created

 

Singleton

 

It’s a type of object that can only be instantiated once.

 

 

 

Prototype

 

It’s just a fancy word of clone.

 

 

 

It’s creates a flat prototype chain that makes it much easier to share functionality between object.

imagine you’re running a hot dog stand and customer places an order.  They need to tell you everything they want in the sandwich.Step-by-step.

Behavioral patterns

- how object communicate with each other

 

Iterator

 

iterator pattern allows you to traverse through a collection of objects

 

 

 

Observer

 

It’s a push-based system. it allows many object to subscribe to events that are boardcast by another object.  It’s a one-to-many relationship.

 

 

 

 

Mediator

 

It’s like a middleman or broker.  Many-to-many relationship.

 

 

 

 

State

 

where an object behaves differently based on a finite number of state.
