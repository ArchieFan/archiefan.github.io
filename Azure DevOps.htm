1 - Full Form of DevOps

 

Dev Ops (software developer and Operations / Admin) is the collaboration of the development and the operations team.  Both development and Operation team should work together hand-in-hand.  DevOps involved three steps.

 

P - philosophy - team have to agree philosophically that we both have to work together. they shouldn't be doing the blame game.

P - process - we should have a process to capture the issues. what has to be done right so if there is a problem during the operations.  there should be some kind of a process which is involved you know so that this communication this cross communication.

T - Tools - rather than having the release manually let us try to use some kind of an automation out here to release it

 

 

 

 

 

 

Note: It is very important is the philosophy that we have to work together.

3 - Defining DevOps

 

DevOps is trying to improve the relationship between the developers and operations.

 

DevOps is a kind of philosophy or kind of a concept or kind of a software development phrase it is not a process, there is no process in DevOps.  It is a working culture which to make the relationship between the developers and the operations department more agile.

 

DevOps is trying to improve working culture between the Developers and Operations Department to make the relationship better so we can have a better, stable and faster delivery.

 

To implement DevOps the first thing is the mindset the working culture.

 

We need to agree that we are all humans, humans work in DEV department and the OPS department. Humans always have problem when it comes to collaborations.

 

There are egos, office politics and blame game and so on.

 

As a human we need to accept that DEV and the OPS departments have issues. We need to resolve it, work as one team so we can deliver the application is a better way to end customers.

 

1)  Decide we want to work in collaborative manner

2)  Operation

5 - DevOps Continuous collaboration

 

 

 

 

DevOps is a working culture which we need to work collaboratively and continuously.

 

1)  The Build should happen automatically

2)  Lot of test cases 80 or 85% is done automated.  we have wrote the unit test case and it will starts firing in the new build. 

3)  The release and deploy if possible it should again an automated, it goes and puts the deployment into a QA server or into UAT server.

4)  From the UAT the operations and the end users start testing If there are issues we again give it back.

 

DevOps does not say we want 100% automation. We cannot replace humans, we can't say will check in the code then build will start automatically the deploy will happen, test will happen.

 

DevOps is all about working culture and the maximum automation that we can do.
